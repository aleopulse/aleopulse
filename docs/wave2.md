## Features Released This Wave

| Feature | Description |
|---------|-------------|
| Anonymous privacy mode | Vote + identity fully hidden via ZK |
| Semi-private privacy mode | Identity public, vote choice hidden |
| Privacy mode selector | Choose privacy level when creating poll |
| Privacy education UI | Explains what each mode means |
| VoteReceipt display | Show proof of participation |
| Vote verification flow | Verify your vote was counted |
| Shield Wallet support | Second wallet option |
| Transaction feedback | Improved status updates |

**Cumulative:** All Wave 1 features + Wave 2 features

## Plan

### Goals
Enhance privacy UX and enable all three privacy modes.

### Planned Deliverables
- [ ] Anonymous privacy mode (vote + identity hidden)
- [ ] Semi-private privacy mode (identity public, vote hidden)
- [ ] Privacy mode selector in poll creation
- [ ] Privacy mode comparison/education component
- [ ] VoteReceipt display and export
- [ ] Vote verification flow
- [ ] Shield Wallet integration
- [ ] Improved transaction feedback

### Success Metrics
- Users understand privacy implications before voting
- All three privacy modes work correctly
- Vote verification is intuitive
- Both wallets connect reliably
